# Mirror this site — 30-second guide

CISR is built to survive any single host going down. The actual content lives in BitTorrent magnets; this site is just the index. Run a mirror in two minutes:

## Quick start (no command line)

1. **Click "Use this template"** at the top of the [source repo](https://github.com/) on GitHub.
2. Name your fork anything (the repo can be private or public).
3. In your new repo, go to **Settings → Pages**.
4. Under "Build and deployment", set **Source: Deploy from a branch**, **Branch: `gh-pages`** (root). Save.
5. Click the **Actions** tab and enable workflows for the fork (one-click button on first visit).
6. The build will run automatically on the next push to `main`. To trigger it now without editing anything: Actions → **Build & deploy static mirror** → **Run workflow**.
7. Wait ~1 minute. Visit `https://<your-user>.github.io/` (or `…/<repo-name>/` if it's a project page).

You are now running a CISR mirror.

## Custom domain (optional)

1. Add a file named `CNAME` at the repo root containing your domain on one line (e.g. `cisr.example.org`).
2. Add `'url' => 'https://cisr.example.org'` to `site/config/config.php` so canonical/hreflang tags reflect the new host.
3. Configure DNS at your registrar: `CNAME cisr → <your-user>.github.io`.
4. In Settings → Pages, enter the custom domain and tick "Enforce HTTPS".
5. Re-run the Action. Done.

## What gets deployed

The `gh-pages` branch is a flat HTML tree generated by `bin/generate.php`. You can also clone it directly:

```bash
git clone --branch gh-pages --depth 1 <repo-url> cisr-mirror
cd cisr-mirror
python3 -m http.server  # local preview
```

Or push the contents to **any** static host (Netlify, Cloudflare Pages, S3, a USB stick, IPFS). The site needs HTTPS for WebTorrent to find browser peers; everything else is plain HTML/CSS/JS.

## Updating from upstream

If you want your mirror to track upstream content:

```bash
git remote add upstream <upstream-url>
git fetch upstream
git merge upstream/main
git push
```

The Action rebuilds and redeploys on push.

## Listing your mirror

If you'd like your mirror visible in every other mirror's sidebar (cross-discovery), open a PR adding a line to `MIRRORS.md`:

```
- [your-domain](https://your-domain.example/) — short note
```

That's the whole protocol.
